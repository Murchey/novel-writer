# 小说写手多智能体系统

## 系统概述
本系统是一个基于多智能体协作的小说创作平台，通过四个专业智能体的协同工作，完成从创意到成稿的完整创作流程。

## 智能体架构

### 1. 灵感工程师（Inspirer）
- **职责**：创意发散，将用户输入拓展为丰富素材
- **输入**：用户的创意需求（主题、关键词、情感基调等）
- **输出**：灵感报告
- **输出路径**：`WORKSPACE/[书名]/AGENT/Inspirer.md`
- **定义文件**：`AGENTS/Inspirer.MD`
- **核心技能**：`SKILLS/Creative_Divergence.MD`

### 2. 章节内容架构师（Architect）
- **职责**：结构设计，将灵感转化为大纲和细纲
- **输入**：灵感报告
- **输出**：全书大纲 + 章节细纲
- **输出路径**：
  - 全书大纲：`WORKSPACE/[书名]/OUTLINE/outline.md`
  - 章节细纲：`WORKSPACE/[书名]/DETAILED_OUTLINE/chapter_[X].md`
- **定义文件**：`AGENTS/Architect.MD`
- **核心技能**：`SKILLS/Structure_Design.MD`

### 3. 写手（Writer）
- **职责**：内容创作，根据细纲完成章节写作
- **输入**：章节细纲 + 风格要求
- **输出**：章节内容
- **输出路径**：`WORKSPACE/[书名]/CONTENT/chapter_[X].md`
- **定义文件**：`AGENTS/Writer.MD`

### 4. 语言润色师（Polisher）
- **职责**：语言优化，提升内容的文学性和人性化
- **输入**：章节内容初稿
- **输出**：润色后的最终版本
- **输出路径**：`WORKSPACE/[书名]/CONTENT/chapter_[X].md`（覆盖原文件）
- **定义文件**：`AGENTS/Polisher.MD`

## 工作流程

### 类型一：全新创作流程
```
用户输入创意需求 → 灵感工程师 → 章节内容架构师 → 写手 → 语言润色师 → 最终输出
```

### 类型二：续写未完成作品流程
当用户提供未完成的小说章节或多章内容要求续写时：

```
用户提供未完成内容 → 需求确认 → 风格提取 → 章节规划 → 大纲细纲 → 章节创作 → 语言润色 → 最终输出
```

#### 详细步骤
1. **需求确认阶段**
   - 询问用户目标章节数量（总章节数）
   - 确认续写方向和结局预期
   - 了解用户对风格的特殊要求
   - 确认已完成章节和待完成章节范围

2. **风格提取阶段**
   - 分析用户提供的未完成内容
   - 提取语言特点、句式风格、用词习惯
   - 提取人物语言风格、叙述视角、节奏特点
   - 生成风格报告，保存到 `WORKSPACE/[书名]/USERSTYLE/style.md`

3. **结构设计阶段**
   - 基于已有内容推断故事走向
   - 设计全书大纲（包含已完成和待完成部分）
   - 编写待完成章节的细纲（按10章分组）
   - 输出大纲到 `WORKSPACE/[书名]/OUTLINE/outline.md`
   - 输出细纲到 `WORKSPACE/[书名]/DETAILED_OUTLINE/chapter_[X].md`

4. **内容创作阶段**
   - 按照细纲进行章节创作
   - 严格遵循提取的用户风格（参考USERSTYLE/style.md）
   - 保持与已有内容的连贯性
   - 输出到 `WORKSPACE/[书名]/CONTENT/chapter_[X].md`

5. **语言润色阶段**
   - 重点消除AI写作痕迹
   - 确保风格与用户原文一致
   - 使语言更自然、更人性化
   - 覆盖原文件输出最终版本

### 类型三：断篇续写流程（已启动项目）
当用户要求续写已启动的项目时：

```
读取进度 → 读取上下文 → 读取细纲 → 自动续写 → 语言润色 → 最终输出
```

#### 详细步骤
1. 读取 `WORKSPACE/[书名]/CURRENT_PROCESS/current.md` 获取进度
2. 读取最近2章内容作为上下文
3. 读取下一章细纲
4. 基于上下文完成续写
5. 语言润色输出最终版本

### 类型一：全新创作详细步骤
1. **创意阶段**
   - 用户提供创意需求
   - 灵感工程师进行创意发散
   - 输出灵感报告

2. **设计阶段**
   - 章节内容架构师接收灵感报告
   - 设计全书大纲
   - 编写章节细纲（长篇按10章分组）

3. **创作阶段**
   - 写手接收章节细纲
   - 阅读风格要求
   - 完成章节内容创作

4. **润色阶段**
   - 语言润色师接收初稿
   - 进行语言优化
   - 输出最终版本

## 输入类型识别

系统根据用户输入内容自动识别输入类型，并触发对应的工作流程：

### 输入类型一：创意需求输入
- **触发条件**：用户提供主题、关键词、情感基调、故事设定等创意要素
- **识别特征**：输入为抽象描述，不含具体章节内容
- **对应工作流**：类型一（全新创作流程）
- **处理流程**：灵感工程师 → 章节内容架构师 → 写手 → 语言润色师 → 最终输出

### 输入类型二：未完成章节续写输入
- **触发条件**：用户提供未完成的小说章节或多个章节内容，要求续写至完结
- **识别特征**：输入包含已有章节正文（至少1章未完结内容）
- **对应工作流**：类型二（续写未完成作品流程）
- **处理流程**：
  1. **询问完结章节数量**：确认用户期望的全书总章节数量、续写方向和结局预期
  2. **提取用户语言风格**：分析未完成内容的语言特点（句式风格、用词习惯、人物语言、叙述视角、节奏特点），生成风格报告保存到 `WORKSPACE/[书名]/USERSTYLE/style.md`
  3. **完成大纲与细纲**：基于已有内容推断故事走向，设计全书大纲（`WORKSPACE/[书名]/OUTLINE/outline.md`），编写待完成章节细纲（`WORKSPACE/[书名]/DETAILED_OUTLINE/chapter_[X].md`），按10章分组
  4. **按细纲完成章节写作**：严格遵循提取的用户风格，保持与已有内容的连贯性，输出到 `WORKSPACE/[书名]/CONTENT/chapter_[X].md`
  5. **语言润色去除AI味**：消除AI写作痕迹，确保风格与用户原文一致，使语言更自然、更人性化，覆盖原文件输出最终版本

### 输入类型三：断篇续写输入
- **触发条件**：用户要求续写已启动的项目（WORKSPACE中已有项目文件夹）
- **识别特征**：输入为项目名称或续写指令，无新章节内容
- **对应工作流**：类型三（断篇续写流程）
- **处理流程**：读取进度 → 读取上下文 → 读取细纲 → 自动续写 → 语言润色

### 输入类型四：章节修订输入
- **触发条件**：用户提供已完成章节内容，要求优化润色或修改
- **识别特征**：输入包含完整章节内容，附带修订/润色指令
- **对应工作流**：语言润色师单独处理
- **处理流程**：读取章节 → 分析修订需求 → 语言优化 → 输出修订版本

## 小说类型支持

### 短篇小说（1-10章）
- 单一流程完成
- 一次性规划所有章节

### 中篇小说（11-30章）
- 分批次规划
- 每批次10章

### 长篇连载（30章以上）
- 分组管理，每10章为一组
- 进度跟踪，记录在CURRENT.MD
- 断篇续写支持

## 进度管理

### CURRENT.MD文件
- 记录当前创作进度
- 包含最近完成章节信息
- 提供续写指引

### 断篇续写流程
1. 读取CURRENT.MD获取进度
2. 读取最近2章内容
3. 读取下一章细纲
4. 基于上下文续写

## 风格管理

### 风格文件
- `STYLES/Default_Style.MD`：默认风格
- `STYLES/Romance_Style.MD`：言情风格
- `STYLES/Mystery_Style.MD`：悬疑风格
- `STYLES/Historical_Style.MD`：历史风格
- `STYLES/Fantasy_Style.MD`：玄奇风格

### 风格选择
- 根据小说类型选择对应风格
- 可自定义风格要求
- 全书保持风格统一

## 使用说明

系统会根据用户输入内容自动识别输入类型（参见"输入类型识别"章节），触发对应工作流程。也可手动指定：

### 类型一：全新创作
1. 告诉系统你的创意需求（主题、关键词、情感基调等）
2. 系统自动启动灵感工程师
3. 按流程完成创作

### 类型二：续写未完成作品
1. 提供未完成的小说章节或多章内容
2. 系统询问目标章节数量和结局预期
3. 系统提取用户风格，保存到USERSTYLE目录
4. 系统设计大纲和细纲
5. 系统按细纲创作，遵循用户风格
6. 系统润色语言，消除AI痕迹

### 类型三：断篇续写（已启动项目）
1. 告诉系统要续写的作品名称
2. 系统读取CURRENT_PROCESS/current.md
3. 系统读取最近2章内容
4. 系统自动完成续写

### 类型四：章节修订
1. 提供已完成的章节内容
2. 说明修订/润色需求
3. 系统进行语言优化，输出修订版本

## 风格管理

### 预设风格文件
- `STYLES/Default_Style.MD`：默认风格
- `STYLES/Romance_Style.MD`：言情风格
- `STYLES/Mystery_Style.MD`：悬疑风格
- `STYLES/Historical_Style.MD`：历史风格
- `STYLES/Fantasy_Style.MD`：玄奇风格

### 用户风格提取（续写时使用）
- 位置：`WORKSPACE/[书名]/USERSTYLE/style.md`
- 内容：从用户提供的未完成内容中提取的风格特点
- 包含：句式风格、用词习惯、人物语言特点、叙述视角、节奏特点等

### 风格选择优先级
1. 用户提供的未完成内容风格（续写时）
2. 用户指定的风格类型
3. 根据小说类型选择对应风格
4. 默认风格

## 注意事项
1. 长篇连载建议按10章分组管理
2. 每完成10章更新进度记录
3. 续写前务必读取最近2章内容
4. 保持全书风格统一
5. 注意情节连贯性和人物一致性
6. 续写时严格遵循用户原文风格
7. 重点消除AI写作痕迹，使语言更自然
